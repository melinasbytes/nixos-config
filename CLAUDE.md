# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A NixOS system configuration for a MacBook (`melbook`), tracked in git and symlinked from `/etc/nixos/`. The two files here are:

- `configuration.nix` — the main system config (edit this)
- `hardware-configuration.nix` — auto-generated by `nixos-generate-config`, do not edit manually

`/etc/nixos/` contains symlinks pointing back to this directory, so edits here take effect when the system is rebuilt.

## Applying changes

```bash
sudo nixos-rebuild switch --flake /home/mel/nixos-config#melbook
```

Test without making the new generation the default boot entry:

```bash
sudo nixos-rebuild test --flake /home/mel/nixos-config#melbook
```

Build only (sanity check, no activation):

```bash
sudo nixos-rebuild build --flake /home/mel/nixos-config#melbook
```

**Important:** Flakes only read git-tracked files. Before rebuilding after adding new files, run `git add <file>` (no commit needed).

## Key architecture notes

**Broadcom WiFi** — This MacBook uses a Broadcom BCM chip with no open-source driver. The `broadcom_sta` proprietary driver is loaded as `wl`, and `b43`/`bcma`/`brcmsmac`/`brcmfmac` are blacklisted to prevent conflicts. The package is currently marked insecure and whitelisted via `nixpkgs.config.permittedInsecurePackages`. Do not remove these blocks or WiFi will stop working.

**Desktop** — Currently GNOME via GDM (temporary, being replaced). Keyboard layout `de` (German), locale mix: `en_US.UTF-8` for UI, `de_DE.UTF-8` for regional formats.

## Target desktop vision (ricing goal)

Building a macOS-inspired, keyboard-driven desktop on top of Hyprland. Reference wallpaper: white sakura / cherry blossom (3840×2160), all colors derived from it.

**Chosen stack:**

| Component | Tool |
|---|---|
| Window Manager | Hyprland (Wayland) |
| Top bar | Waybar — slim, macOS-style, top edge only |
| Launcher | anyrun — Raycast-style text input for apps + settings |
| Color theming | Stylix — derives base16 palette from wallpaper, applies system-wide |
| Dotfile management | Home Manager |
| Notifications | mako |

**Design constraints:**
- No Windows-style taskbar; only a slim top bar with WiFi/LAN/BT, clock/date, power
- Multi-monitor support with easy keyboard-driven window movement
- Everything declared in Nix — no manual dotfile edits outside the repo

**Build order:**
1. Home Manager setup
2. Stylix + wallpaper → color palette
3. Hyprland alongside GNOME (switchable at GDM)
4. Waybar + anyrun
5. Remove GNOME once everything works

**Audio** — PipeWire with ALSA and PulseAudio compatibility; `services.pulseaudio.enable` must stay `false`.
